# Инструкции по завершению портирования Tango из WP7 в UWP

## Обзор проделанной работы

Приложение Tango было успешно адаптировано для работы в среде UWP. Ниже приведены ключевые изменения, которые были выполнены:

1. Обновлены зависимости проекта в WinPhoneTango.csproj:
   - Заменена версия Microsoft.UI.Xaml с 2.8.6 на 2.7.0 для совместимости с целевой платформой
   - Обновлена TargetPlatformMinVersion до 10.0.17763.0

2. Исправлены артефакты WP7/Silverlight:
   - Удалены устаревшие ссылки на Microsoft.Phone.Controls и Microsoft.Phone.Shell
   - Заменены устаревшие элементы управления на UWP-совместимые аналоги
   - Исправлены пространства имен, связанные с Windows Phone

3. Обновлены XAML-файлы страниц:
   - Исправлены пути к ресурсам (изменены с "/WinPhoneTango;component/" на "ms-appx:///WinPhoneTango/")
   - Обновлены стили и шаблоны элементов управления для соответствия UWP
   - Убраны устаревшие атрибуты, специфичные для WP7

4. Обновлены классы-конвертеры:
   - Исправлены пространства имен и зависимости для IValueConverter
   - Обновлены ссылки на UWP-совместимые типы данных

## Шаги для завершения портирования

### 1. Проверка и исправление оставшихся зависимостей

Некоторые зависимости могут требовать дополнительных изменений:

- Проверьте, что все ссылки на `Microsoft.Phone.*` пространства имен полностью удалены
- Убедитесь, что все элементы управления, такие как `Pivot`, `LongListSelector`, `ApplicationBar`, заменены на UWP-совместимые аналоги
- Замените любые оставшиеся `System.Windows.*` на `Windows.UI.Xaml.*`

### 2. Адаптация функций, специфичных для WP7

Некоторые функции могли остаться неперенесенными:

- Проверьте использование `PhotoCamera` и других камеро-специфичных API - они должны быть заменены на UWP-совместимые
- Убедитесь, что все фоновые задачи адаптированы для UWP
- Проверьте работу с контактами и телефонной книгой - API могли измениться

### 3. Тестирование сборки

Запустите полную сборку проекта:

```bash
MSBuild.exe "Tango.sln" /p:Configuration=Debug /p:Platform=x64 /t:Build
```

Если возникнут ошибки компиляции:
- Проверьте, что все зависимости правильно установлены
- Убедитесь, что все устаревшие ссылки заменены
- Проверьте, что все необходимые NuGet-пакеты обновлены

### 4. Решение проблем с совместимостью

Если при запуске приложения возникают ошибки:
- Проверьте, что все XAML-разметки корректны
- Убедитесь, что все ресурсы доступны по новым путям
- Проверьте, что все стили и темы совместимы с UWP

### 5. Финальное тестирование

После успешной сборки протестируйте основные функции приложения:
- Запуск и навигация между страницами
- Работа аудио/видео вызовов (если применимо)
- Работа с контактами и сообщениями
- Корректная работа UI-компонентов

## Возможные проблемы и решения

1. **Ошибки при сборке**: Убедитесь, что TargetPlatformMinVersion >= 10.0.17763.0 и что все зависимости обновлены

2. **Отсутствующие ресурсы**: Проверьте, что все пути к ресурсам начинаются с "ms-appx:///" и не содержат старых форматов

3. **Проблемы с отображением**: Убедитесь, что все стили и шаблоны элементов соответствуют UWP-требованиям

4. **Отсутствующие API**: Замените любые устаревшие WP7-специфичные API на UWP-совместимые аналоги

## Заключение

После выполнения всех вышеуказанных шагов ваше приложение должно корректно работать в среде UWP. Убедитесь, что все основные функции протестированы и приложение стабильно работает.
